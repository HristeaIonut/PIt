1 Frontend

We started the page by looking out for different styles, selecting the fonts and representative images; We consulted each other it order to make sure everything is clear on the decisions and that we all agree on specified themes.

The first page was the main one. There were no functionalities yet, but we tried to make it as attractive as possible.

We chose a dark theme, as lately more users are starting to incline towards it. It is not as tiring for the eyes as a light one.

Next up, we created the other pages: Contact, How to use, Report.

After we started working on the backend as well, we noticed that we needed to add a new page, called "My Pastes"

We also created different html files for when a user is logged in, so that his username appears on the right upper side of the screen, as well as the option to log out.

2 Backend

The first thing we did when we started working on the backend side was to take the code inserted by the user and create a new file using it; The user is then redirected to a newly created page containing his code. We did that by creating a template, which will be filled every time with information provided by the user.

Next up, we made the database where we would add user accounts, containing their name, email adress, username and password. We added the php script which connects our code to the database in order to collect information added by the user. Afterwards, we handled the login page, where an user can add hid already existing information in order to enter his account.

Once we were done with this, our main goal was to work on the functionalities of our website. The first thing was the syntax highlighting, for which we worked in javascript and used regex to select words that we wanted highlighted, as provided by each programming language.

Afterwards, we created the option to insert a password while creating a paste. This information is also added to the database and it will be asked for from any person trying to access the code, except for the author.

While two of us were focused on these main functionalities, a third person was designing the different pages for authenticated and non-authenticated users and adding the required options there as well.

The next step was to add the "burn after read" option, which would erase a paste after a single read. The code will delete the paste as soon as the person reading it will exit the page.

The last option we added to the code was to edit it. During the creation process, a user decides wether or not he should be the only one with access to edit the paste. After that, any person who has this right will be able to bring as many changes as he wants, and will alwasy be able to tollback to the previous 3 versions.
