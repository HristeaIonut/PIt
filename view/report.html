1 Abstract

A PasteIt is a Web application that allows users to upload and share text online. The most common use is for sharing source code or configuration information. There are thousands of pastebins online, often geared towards particular groups or focuses. Once text has been uploaded to a pastebin, other users can edit.

2 Introduction

The goal of this project is to offer a platform through which one can share their code among friends, colleagues and even teachers, in a more convenient way. Our app can be used by both authenticated and anonymous user.

For authenticated users:

  • One can manage their own posts(edit, delete previous posts);
  • The option of selecting other users who can modify the posts and keep a history for rolling back to previous versions;
  • Extra options such as "burn after read", "password protected", "expiration date" or "reading confirmation - user will be able to see who read the post".

For anonymous users:

  • The posts can only be public and only after they will have passed the Captcha test;
  • The posts will be kept for a maximum of 30 days.

2.1 Purpose

The purpose of our Website is mainly to make programmers' lives easier. How will that work? When a user wants to paste a code, that person can make it pretty. This means that the code becomes easier to read and therefore, easier to understand. Another advantage is encryption, which allows a user to share it in a secure environment, because we value intellectual property and wouldn't want it to be violated. Our current goal is to let users know that they are being heard, and this is why we will constantly bring changes and updates, according to the received reviews.

2.2 Intended Audience and Reading Suggestions

We expect our website to be used by any person who wants to share their code, wether it is a highschool student or a college student, a teacher or a professor, a hired developper or a private one. As we do not require this kind of information when a new user creates an account, they can freely write whatever code they want.

3.1 Project development stages

  1. Users will paste a code in the container area by pasting it; afterwards, we check if the box was left empty when selecting the option "Create paste"; if that is the case, we let the user know through an alert
  2. They may select a programming language in order to highlight the code's syntax; we did that by creating javascript files for each language, where we used regex in order to select the words which rewuired highlighting. The available languages to highlight are the following:
    • C
    • C#
    • C++
    • HTML
    • CSS
    • JSON
    • Java
    • JavaScript
    • Python
    • Bash
  3. They can select an expiration time for their paste(which is, by default, disabled). This is then added in our database, in a column named "expiration_date"(by adding the selected time to the current timestamp). Whenever a user loads a paste via a link or through his own page "My pastes", the code checks wether or not that time has already passed, for each paste; if it did, those pastes will not be shown or accessible to the user, even though they do remain in the database. The user will have the following options:
    • 5 minutes
    • 10 minutes
    • 30 minutes
    • 1 hour
    • 1 day
    • 1 week
    • 1 month
  4. The author may also choose wether or not the document can be edited by any other user or not; If they select this, then any other user who accesses the paste will have the option to edit it and then rollback to previous versions. If this box will not be ticked, then only the author can make changes to the paste. The rollback will work in the exact same way.
  5. There will be an option to "burn after read", which means that if the person who was intended to read the document will have read it, it will be erased.
  6. The option to insert a password as a security measurment, so that only users who have it can view the document. The password is then inserted into the database in a hashed version, which is being checked whenever someone wiches to have access to the paste. If it is wrong, clearly, they won't.

3.2 Structure

This app is structured in 5 "subpages";

  • The main one is the Paste page, where all the magic happens. One may share their code, edit or view someone else's.
  • We have a contact page, which helps users reach to an administrator whenever there is a problem, whether or not it regards the code implementation.
  • The "How to Use" page is aimed to teach new users about the goal and the ways of using our app.
  • The "Register" page is meant for new users, so that they can create an account through which they may use the app in the future.
  • The "Login" page will be used for users to enter their already existing account, so they can be able to view they history of what the already pasted, to edit or delete something or rollback to previous changes.

4 Conclusion

Our website is a user friendly environment, which allows them to safely share their work among friends, colleagues and others; We designed a variety of options in order to make the code look good and give the author a chance to dirrectly express how he'd like it to be handled: who can see it, how long it will remain available or who can edit it.
